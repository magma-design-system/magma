The `<mds-progress>` web component is the progress indicator of the Magma Design System, rendering the completion state of a determinate task as a bar or ring. It is the system equivalent of the native HTML `<progress>` element, exposing the appropriate `progressbar` role and ARIA value attributes natively.

#### Semantic Behavior

- **Accessibility role**: Renders with `role="progressbar"` and fixed `aria-valuemin="0"` / `aria-valuemax="100"`; `aria-valuenow` is derived by rounding `progress * 100`.
- **Spoken step text**: As `progress` changes, the component maps the value onto the `steps` list and writes the matching label to `aria-valuetext`, so assistive technologies announce a human-readable milestone rather than a raw number.
- **Decorative opt-out**: When the host carries `aria-hidden="true"`, all progressbar semantics (`role`, the `aria-value*` attributes) are stripped so the indicator can be used as purely decorative chrome.
- **Reactive updates**: The `progress` and `steps` props are watched; changing either at runtime recomputes the current step and the announced value without re-mount.
- **Radial delegation**: With `direction="radial"` the rendering is delegated to a child `mds-radial-progress`, which carries the typography and the textual percentage; the bar directions render a single fill `div` instead.

#### Properties & Visual Configurations

- **`progress`** is the source of truth: a fractional value between `0` and `1` (not a percentage) that drives both the visual fill and the computed ARIA value.
- **`steps`** is a comma-separated string of milestone labels (default Italian: `'Inizio,Un quarto,Metà,Tre quarti,Fine'`); the current `progress` is quantized across these labels to pick what screen readers announce. Provide labels that match the granularity of the task.
- **`typography`** (`'option'` | `'label'`) selects the technical type scale applied to the textual percentage shown inside the radial variant; it has no effect on the bar directions.

#### Other behavioral props

- **`direction`** chooses the geometry: `'horizontal'` and `'vertical'` render a linear fill bar, while `'radial'` renders a circular ring with a centered percentage via `mds-radial-progress`.

The shared `variant` and `size` ladders are defined in [`projects/stencil/SPEC.md`](../../../../SPEC.md#tone-and-variant-system); this component adds no values beyond the shared sets.
