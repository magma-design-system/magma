The `<mds-price-table-list-item>` web component represents a single feature row inside a [`<mds-price-table-list>`](../../mds-price-table-list), pairing a supported/unsupported status icon with a textual feature label. It is a presentational compound child that replaces a plain list item (`<li>`-like) row within the parent's feature list.

#### Semantic Behavior

- **Compound child only**: It auto-assigns `slot="item"` on its Host, so it is captured exclusively by the parent's `<slot name="item" />`. It must be a direct child of `<mds-price-table-list>` and is never used standalone or mixed with the parent's other named slots (`header`, `price`, `action`).
- **Drives parent layout**: The parent counts elements matching `[slot="item"], mds-price-table-list-item` on load; the presence of at least one item is what makes the parent render its separator and content region. Items added after the parent's `componentWillLoad` are not retroactively detected.
- **Status icon by `supported`**: When `supported` is `true` it renders a filled check-circle icon; when `false` it renders a horizontal-rule (dash) icon, indicating the feature is absent. The `supported` value is reflected to the host attribute for styling.
- **Default slot is the label**: Text or markup placed in the default slot becomes the feature description, rendered next to the icon inside an `mds-text` wrapper.
- **No interactivity**: The component is purely presentational — it has no role, selection/active/disabled state, emitted events, or keyboard handling.

#### Properties & Visual Configurations

- **`supported`**: Choose `true` for features included in the plan (check icon) and `false` for features not included (dash icon). It is the only semantic switch that changes the rendered icon, and is reflected for theming via the `--mds-price-table-list-item-supported-*` / `-unsupported-*` CSS custom properties.
- **`typography`**: Selects the text scale applied to both the icon wrapper and the label, accepting one of the shared read typography tokens (default `detail`). Use a larger value such as `paragraph` when the list should read at body size, or `caption` for denser tables.
