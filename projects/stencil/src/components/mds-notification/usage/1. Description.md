The `<mds-notification>` web component is a numeric badge/dot indicator of the Magma Design System that attaches to another element (an icon, a button, an avatar) to signal a count of pending items. It has no HTML primitive equivalent: it positions itself relative to a target element rather than wrapping content of its own.

#### Semantic Behavior

- **Target attachment**: The component locates its anchor by running `querySelector` against the `target` selector and positions itself against that element using floating-ui (`computePosition` + `autoUpdate`), tracking the anchor as it scrolls or resizes.
- **Missing target falls back**: If `target` is not set, the component forces `strategy` to `'disabled'` and skips positioning; if `target` is set but resolves to no element, it throws `No valid target found` at render time.
- **Zero-value hiding**: When `value` is `0` the dot renders empty, so the badge effectively disappears until there is a count to show.
- **Count overflow**: When `max` is set and `value` exceeds it, the displayed text becomes `+max` (e.g. `+9`) instead of the raw number; otherwise the value is locale-formatted via `toLocaleString()`.
- **Accessibility**: The host exposes `aria-labelledby` pointing at the `target` and an `aria-label` carrying the current value, while the inner count text is `aria-hidden` to avoid double announcement.
- **Lifecycle cleanup**: The floating-ui auto-update subscription is torn down on disconnect and whenever `strategy` switches to `'disabled'`.

#### Properties & Visual Configurations

- **`target`** is the CSS selector (passed to `querySelector`) of the element the badge anchors to; it is effectively mandatory for positioned rendering.
- **`value`** is the count shown in the badge and also the source of the accessible label; `0` hides the badge.
- **`max`** caps the visible number — pick it when raw counts could grow large and you want a compact `+N` ceiling.
- **`visible`** toggles whether the badge is shown independently of its value.

#### Other behavioral props

- **`strategy`** selects the floating-ui positioning mode (`'fixed'` by default, `'absolute'`, or `'disabled'`). Use `'disabled'` to opt out of dynamic tracking; it is also set automatically when no `target` is provided.
