The `<mds-push-notification-item>` web component is a single notification card rendered inside its parent container [`<mds-push-notification>`](../../mds-push-notification). It composes an avatar/image, a subject, a relative or formatted timestamp, a message body and an optional dismiss control into one toast-like row; it is not a native HTML primitive but a structured content card meant to be stacked within the parent's animated notification area.

#### Semantic Behavior

- **Compound child only**: It must be placed as a direct default-slot child of `<mds-push-notification>`; it is never used standalone or mixed with other child types, since the parent reads each item from its slot to animate intro/outro and manage stacking.
- **Dismiss reports up, removal happens in the parent**: Clicking the built-in close button emits the bubbling `mdsPushNotificationItemClose` event (stopping propagation of the raw DOM click first); the parent listens for it, runs the outro animation and removes the host from the DOM. The item never removes itself.
- **Parent-driven visibility lifecycle**: When the last item closes, the parent auto-hides the whole notification area, so an item's dismissal indirectly drives parent state.
- **Timestamp rendering**: `datetime` is sanitized to ISO 8601 on load and rendered through dayjs; with `dateFormat="timeago"` it shows a localized relative time ("2 minutes ago"), otherwise the value is treated as a dayjs format string for a static date.
- **Localization**: The relative-time strings and the dismiss button title are localized (el/en/es/it), resolved from the host's language; `updateLang()` re-resolves the active language at runtime.
- **Conditional avatar vs. picture**: An `<mds-avatar>` is rendered when `icon` is set or `preview="avatar"`; when `src` is set and `preview` is not `avatar`, a full-width `<mds-img>` preview is rendered instead.
- **Slot detection**: The `badge` and `action` named slots are only wired into the layout when matching slotted children exist at load, so empty slots add no markup.

#### Properties & Visual Configurations

- **`deletable`** (default `true`): Controls whether the dismiss button is shown. Keep it on for user-removable notifications; set `false` for items the user should not close manually. Note a normalization quirk — setting it to `false` resets the internal value to `undefined`, which simply suppresses the close button.
- **`preview`**: Pick `avatar` to show the `src` image (or `initials`/`icon` fallback) as a compact round avatar, or `image` (default) to show `src` as a larger inline picture preview.
- **`initials`**: Provided as the avatar fallback when no image is available; it intentionally overrides `tone`/`variant` styling so the user stays visually recognizable.
- **`dateFormat`**: Use `timeago` for a live relative timestamp, or any dayjs format token string for a fixed display date.

For `tone` and `variant`, this component consumes the shared color ladders defined in [`projects/stencil/SPEC.md`](../../../../SPEC.md#tone-and-variant-system); the values are forwarded to the underlying `<mds-avatar>` and follow the avatar tone/variant set rather than adding component-specific values.
