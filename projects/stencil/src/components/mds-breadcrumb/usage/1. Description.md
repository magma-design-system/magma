The `<mds-breadcrumb>` web component is the navigation-trail container of the Magma Design System. It is a compound parent that arranges a sequence of slotted `<mds-breadcrumb-item>` children into a hierarchical path and adds an optional back-navigation control.

#### Semantic Behavior

- **Compound parent/child**: The default slot accepts only `<mds-breadcrumb-item>` children; on load each child is assigned a sequential `item-N` id that drives selection and back-stepping logic.
- **Selection tracking**: Listens for the child `mdsBreadcrumbItemSelect` event, then marks exactly the emitting item as `selected` and clears the others, so a single depth is always current.
- **Back button**: When `back` is enabled the host renders a leading `<mds-button>` arrow that steps selection to the previous item; it auto-disables (and detaches its keyboard handler) whenever the first item is current.
- **Change event**: Emits `mdsBreadcrumbChange` after any selection change — via a child click or the back arrow — carrying the new index `id` and the originating `caller` item.
- **Keyboard interaction**: The back control is wired through the shared `KeyboardManager`, so it activates on click-equivalent keys and is removed from interaction when disabled.
- **Localization**: The back button's `title` is resolved per document language (el/en/es/it) and can be refreshed at runtime via the `updateLang()` method.

#### Properties & Visual Configurations

This component exposes a single behavioral prop:

- **`back`** toggles the leading arrow control. Leave it enabled (the default) for multi-level trails where users benefit from a one-tap step backwards; disable it for shallow or display-only breadcrumbs where reverse navigation adds no value.

Visual styling (button colors, current-depth color, separator arrow color) is driven entirely by the CSS custom properties documented in [`readme.md`](../readme.md), not by props. The shared `variant` / `tone` / `size` ladders defined in [`projects/stencil/SPEC.md`](../../../../SPEC.md#tone-and-variant-system) do not apply here; per-item labels and selection state live on the `<mds-breadcrumb-item>` children.
