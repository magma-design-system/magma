The `<mds-pref-theme-variant>` web component is the theme-switcher segment of the Magma accessibility preferences panel, designed to live as a direct child of [`<mds-pref>`](../../mds-pref). It renders a labelled tab trigger that opens a dropdown of selectable theme variants and, when a variant is chosen, applies it globally by mutating the document's `<html>` element (theme name class, scheme class, `data-theme-name` attribute and custom properties).

#### Semantic Behavior

- **Compound positioning**: It must be a direct slot child of `<mds-pref>` and is never used standalone or mixed with the other preference segments (`mds-pref-animation`, `mds-pref-contrast`, etc.) as siblings of a different parent.
- **Acts as a sub-parent itself**: Its own default slot must contain one or more `mds-pref-theme-variant-item` elements; it queries them, wires their `mdsPrefThemeVariantItemSelect` events, and drives their `selected` state so only the active variant is marked selected.
- **Global theme application**: Selecting an item applies the theme document-wide — it toggles `pref-theme-scheme-*` and `pref-theme-name-*` classes on `<html>`, sets `data-theme-name`, and writes the `--magma-pref-theme-name` / `--magma-pref-theme-scheme` custom properties.
- **Persistence and hydration**: The current name and scheme are stored in `localStorage` (`mdsPrefThemeName` / `mdsPrefThemeScheme`) and re-read on every render, so a previously chosen variant is restored across reloads regardless of the declared `name` default.
- **Name validation**: An invalid theme name (not lowercase kebab-case matching `^[a-z]+(-[a-z]+)*$`) throws at runtime when the variant is applied.
- **Events bubbled up**: Emits `mdsPrefThemeVariantChange` with the `{ name, scheme }` detail on selection, and `mdsPrefChange` (`preference: 'theme-variant'`) which `<mds-pref>` listens to in controller mode.
- **Dropdown interaction**: The tab item toggles an internal `mds-dropdown` (closing on selection or outside hide); the chevron icon flips between up and down to reflect the open state.
- **Localized label**: The "Theme" heading is resolved through the shared `Locale` helper and follows the active interface language.

#### Properties & Visual Configurations

- **`name`**: The active theme variant identifier (default `'default'`); it is reflected and mutable, and is overridden at runtime by any persisted `localStorage` value. Use it to declare the initial theme when no preference has been stored yet.
- **`scheme`**: Constrains the variant's colour mode to `'light'`, `'dark'`, or `'all'` (default). Pick `'all'` for themes that ship both light and dark palettes; pick `'light'` or `'dark'` to force a single mode for a variant that only defines one.
- **`size`**: Sets the size (`'sm'` / `'md'`) of the nested tab UI. It is normally not set directly — `<mds-pref>` propagates its own `size` down to every `mds-pref-*` segment so the panel stays visually consistent.
