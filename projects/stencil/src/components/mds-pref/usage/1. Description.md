The `<mds-pref>` web component is the accessibility-preferences panel of the Magma Design System: a compound parent that groups user-facing settings (animation, consumption, contrast, theme, language) so people can tune readability, motion, and language for their session. It follows the `mds-tab` pattern and acts as the coordinator for the `mds-pref-*` children slotted inside it.

#### Semantic Behavior

- **Compound parent**: Renders a single default `<slot>` and expects `mds-pref-animation`, `mds-pref-consumption`, `mds-pref-contrast`, `mds-pref-theme`, and/or `mds-pref-language` children; it owns no preference UI of its own beyond coordination.
- **Size propagation**: Changing `size` cascades the value down to every nested `mds-pref-*` child, keeping the whole panel on one tab size.
- **Reload notice**: Listens for the `mdsPrefChange` event from children and, when the changed preference is one that cannot apply live (`consumption` or `language`), reveals an inline caption prompting the user to refresh the page.
- **Live language sync**: When a `language` change is emitted, the panel re-resolves its own locale so the reload notice is shown in the newly selected language; `updateLang()` exposes this refresh as a public method.
- **Document marking**: On load it sets the `data-magma-pref` attribute on `<html>` (and removes it when disconnected), signalling to the page that the preference system is active.
- **Controller mode wiring**: Child `mdsPrefChange` listeners are only attached when the component runs in controller mode (see below); as a visible UI panel it stays passive until a child reports a change.

#### Properties & Visual Configurations

`size` is one of the shared tab sizes and is forwarded to children rather than styling the host directly — set it once on the parent instead of on each child.

#### Other behavioral props

- **`controller`** switches the component from a visible on-DOM panel into a headless controller that applies and reacts to preferences without rendering UI; in this mode it binds the child change-listeners used to drive document-level state. Setting it to `false` is normalized back to `undefined` (the attribute is removed) so it behaves as a true presence flag.
