The `<mds-pref-animation>` web component is a preference control that lets users choose how animations are presented (reduced, system-driven, or fully enabled). It is a compound child of [`<mds-pref>`](../../mds-pref) and renders a labelled three-option tab group rather than any native HTML primitive.

#### Semantic Behavior

- **Compound child constraint**: Must be placed as a direct default-slot child of `<mds-pref>`; it is never used standalone or mixed into other containers, and `<mds-pref>` discovers it by querying for `mds-pref-*` descendants.
- **Tab-group selection**: Internally renders an `<mds-tab>` with three `<mds-tab-item>` entries (reduce / system / no-preference); the active item is driven by the `mode` value, and clicking an item sets that mode.
- **Global side effects on selection**: Choosing a mode toggles a corresponding class (`pref-animation-reduce` / `pref-animation-system` / `pref-animation-no-preference`) on the document `<html>` element, sets the `--magma-pref-animation` custom property, and persists the choice to `localStorage` under `mdsPrefAnimation`.
- **Initial mode resolution**: On first render the effective mode is resolved in order from the `mode` prop, then the persisted `localStorage` value, then the `system` default.
- **Reflected, mutable mode**: `mode` is reflected to an attribute and mutated internally as the user interacts; a `@Watch` re-applies all side effects whenever it changes.
- **Bubbled change event**: Emits `mdsPrefChange` (detail `{ preference: 'animation' }`) on every selection; the parent `<mds-pref>` listens for this to coordinate cross-preference behavior such as the reload-required hint.
- **Localized label**: Shows a localized "Animations" caption with the current mode's human label, following the locale resolved from the host (`updateLang()` re-reads it).

#### Properties & Visual Configurations

- **`mode`**: Sets which animation preference is active — `reduce` to suppress motion, `no-preference` to fully enable animations, or `system` to defer to the user's OS setting. Leave unset to fall back to the persisted or default (`system`) value.
- **`size`**: Sizes the nested tab items (`sm` or `md`). It is normally not set directly; the parent `<mds-pref>` propagates its own `size` down to every `mds-pref-*` child, so set it on the parent for a consistent group.
