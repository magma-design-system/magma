The `<mds-input-date-range-preselection>` web component is a compound child of [`<mds-input-date-range>`](../../mds-input-date-range). It renders a clickable shortcut button (a quick range like "Last 7 days" or "This month") that, when activated, fills the parent's date range with a predefined `start`/`end` pair instead of requiring the user to pick dates manually on the calendar.

#### Semantic Behavior

- **Compound child only**: Must be placed as a direct child of `<mds-input-date-range>`; it auto-assigns itself to the parent's `calendar-preselection` slot via `<Host slot="calendar-preselection">` and is never used standalone or mixed with other child types.
- **Click drives the parent**: On click it locates its ancestor `<mds-input-date-range>` and invokes that parent's `preselect()` method, passing itself as `caller` along with its `start` and `end`, so the parent populates and validates the range, syncs the form value, and closes the dropdown.
- **Selection is parent-managed**: The `selected` prop is reflected and mutable but is owned by the parent — choosing one preselection clears `selected` on all siblings and marks the active one, and the parent re-evaluates `selected` whenever the current range matches a preselection's dates (including single-day matches where `end` is undefined).
- **Internally a button**: Its visible control is an `<mds-button variant="primary" tone="strong">` wrapping the default `<slot>`, so the label is whatever you slot in; the click handler stops propagation to avoid leaking to calendar/dropdown listeners.

#### Properties & Visual Configurations

- **`start`** (required) and **`end`**: Define the ISO date range this shortcut applies. Provide both for a true range; omit `end` to express a single-day preselection (the parent treats the range as `start` only and matches it accordingly).
- **`selected`**: Reflects whether this preselection is currently the active one. Treat it as read-only in normal use — the parent sets it; do not hand-toggle it to drive selection.

The default slot holds the human-readable label. Visual states (default, hover, selected) are themed through the `--mds-date-range-preselection-*` CSS custom properties listed in `readme.md`; the component does not expose `variant`/`tone` props of its own.
