The `<mds-price-table-list>` web component is a single pricing-plan column inside the [`<mds-price-table>`](../../mds-price-table) layout container. It is a compound child that arranges a plan's heading, feature list, price and call-to-action into a consistent card-like structure built entirely from slots — it has no configurable props.

#### Semantic Behavior

- **Compound child**: Designed to sit as a direct slot child of [`<mds-price-table>`](../../mds-price-table); one instance represents one pricing plan, and several siblings line up to form the comparison table. It is not meant to be used standalone.
- **Owns the feature-list child type**: Feature rows are provided as `<mds-price-table-list-item>` elements, which reflect `slot="item"` onto their host and are projected into the component's internal `item` slot — they are the intended child type for the list body.
- **Conditional list region**: On `componentWillLoad` the component detects whether any `[slot="item"]` / `<mds-price-table-list-item>` children are present and only then renders the feature `main` region and the `<mds-separator>` above it; a plan with no features collapses to header plus footer.
- **No interactive state**: It renders pure structure with no `role`/ARIA defaults, no selected/active/disabled state and no emitted events; any interactivity (e.g. the CTA) comes from the components you slot in, such as `<mds-button>` in the `action` slot.
- **Footer composition**: The `price` and `action` slots are rendered together inside a single `footer` part, so the price and its call-to-action are visually grouped at the bottom of the card.

#### Properties & Visual Configurations

`<mds-price-table-list>` exposes no `@Prop()`s; it is a layout-only child whose appearance is driven entirely by what you slot into it. Use the named slots to compose a plan:

- **`header`**: the plan title and supporting description (typically `<mds-text>` elements); rendered in the `header` part at the top.
- **default body**: the feature rows, supplied as `<mds-price-table-list-item>` children; these populate the optional `content` region.
- **`price`**: the plan's price, shown in the grouped `footer`.
- **`action`**: the call-to-action; `<mds-button>` is recommended.

The only style hook is the `--mds-price-table-list-separator-color` CSS custom property, controlling the divider drawn between the header and the feature list.
