The `<mds-pref-language-item>` web component represents a single selectable language option inside the language-preference picker [`<mds-pref-language>`](../../mds-pref-language). It renders one row of the parent's dropdown as a text `mds-button` with a check/circle indicator, and reports the user's choice up to the parent rather than changing the page language itself.

#### Semantic Behavior

- **Compound child only**: Must be placed as a direct default-slot child of `<mds-pref-language>`; the parent forwards the slot into its internal `mds-dropdown`. It is never used standalone or mixed with other child types — outside its parent nothing listens for its event and selection cannot be coordinated.
- **Code validation at render**: `code` must match an entry in the bundled language dictionary; on `componentWillRender` an unknown `code` throws `Language code not found: <code>`. When `code` is empty the item renders an error-toned fallback button instead of a language label.
- **Label is derived, not authored**: The visible language name is looked up from `code` against the internal dictionary (e.g. `it` to `Italiano`), so consumers supply only the code, not the display text.
- **Selection is parent-driven**: The `selected` prop is owned and toggled by `<mds-pref-language>`, which clears every sibling and sets only the active item. Authors generally do not set it manually; it reflects to an attribute for styling the chosen row.
- **Click emits, does not commit**: Clicking the host emits the bubbling `mdsPrefLanguageItemSelect` event carrying `{ language: code }`. The parent catches it, updates selection state, persists the preference, sets `<html lang>`, and emits its own `mdsPrefLanguageChange` — this child performs none of that.
- **Visual selection indicator**: A selected item shows a filled `check-circle` icon; an unselected one shows an outline `circle`, both rendered through a `dark` / `text` `mds-button`.

#### Properties & Visual Configurations

The only meaningful prop is `code`: the BCP 47 / RFC 5646 language tag (such as `it`, `en`, `es`) that both keys the dictionary lookup for the label and is the payload emitted on selection. The set of accepted codes is defined by the bundled dictionary in `meta/locale.json` — do not enumerate it here.

`selected` is a state flag managed by the parent (see Semantic Behavior) rather than a configuration choice. Sizing and the `text` button styling are fixed by the parent's layout, so this item exposes no `variant` / `tone` of its own; the shared ladders in [`projects/stencil/SPEC.md`](../../../../SPEC.md#tone-and-variant-system) are applied internally and are not configurable from the host.
