The `<mds-table-body>` web component is the row-group container of the Magma Design System tabular data system, used inside [`<mds-table>`](../../mds-table). It is the Stencil equivalent of the native HTML `<tbody>`, grouping the data `mds-table-row` elements and acting as the relay through which the parent table propagates interaction and selection styling to those rows.

#### Semantic Behavior

- **Compound child**: Must be a direct slot child of `<mds-table>` and hold only `mds-table-row` elements; it is never used standalone nor interleaved with `mds-table-header` / `mds-table-footer` content.
- **Row-group role**: Renders `role="rowgroup"` on the host and lays out as `display: table-row-group`, so the semantic table structure is contributed by this element rather than by the parent `role="table"` wrapper.
- **Parent-driven state**: Both `interactive` and `selection` are reflected attributes that the parent table sets imperatively — `interactive` via the table's `interactive` watcher, and `selection` whenever the table recomputes its row selection — so consumers normally do not toggle them by hand.
- **Mutation observed**: The parent attaches a `MutationObserver` to this body to detect rows added or removed at runtime, then re-applies interactive/selectable state to the new rows.
- **Hover relay**: When `interactive` is set, hovering the body overrides the internal `--mds-table-row-background` / row color tokens to the hover palette (excluded on Safari via host-context), driving the row highlight effect for the whole group.
- **Last-row border trimming**: The last `mds-table-body` in a table clears the bottom cell border of its final row, so stacked bodies join visually without a doubled separator.

#### Properties & Visual Configurations

This component exposes only two boolean flags, both intended to be managed by the parent `<mds-table>` rather than set directly:

- **`interactive`** enables the mouseover row-highlight styling for the group; it mirrors the table's own `interactive` prop and is what gives clickable/navigable tables their hover feedback.
- **`selection`** switches the body's default background to its alternate tone to signal that an active selection exists; the parent sets it true while at least one row in the table is selected.
