The `<mds-table-header-cell>` web component is the column-header cell of a Magma data table. It is a compound child of [`<mds-table-header>`](../../mds-table-header) and plays the role of a native `<th>` (`columnheader`), rendering the column label and, when enabled, acting as the interactive sort control for its column.

#### Semantic Behavior

- **Compound child only**: Must be placed as a direct slot child of `<mds-table-header>`; it is never used standalone, and its sorting logic relies on resolving its sibling header cells and the sibling `mds-table-body` rows within the enclosing `mds-table`.
- **Header role and ARIA sort**: The host renders with `role="columnheader"` and reflects its current sort state through `aria-sort`, mirroring the `direction` prop (`none` / `ascending` / `descending`).
- **Position-based sorting**: On load the cell computes its own index among its header-cell siblings and uses that index to read and reorder the matching `mds-table-cell` of every body row; values come from each cell's `value` (falling back to trimmed text), with numeric values compared numerically and others compared via `localeCompare`.
- **Tri-state sort cycle**: Each activation advances the column through `none` → `ascending` → `descending` and back to `none`; the `none` state restores the original (default) row order captured on load.
- **Single active sort column**: Sorting a column resets every sibling header cell's `direction` to `none` and tags only the active column's body cells and sorted rows with a `sorted` attribute, so styling reflects which column drives the order.
- **Conditional rendering**: When `sortable` is set the cell renders an `mds-button` (text tone, with an up / down / unfold-more icon reflecting `direction`) as the clickable sorter; otherwise it renders a plain `mds-text` label with no interaction.
- **Reactive props**: A watcher on `sortable` re-runs sorter preparation, and a watcher on `direction` re-syncs internal state when it is externally set back to `none`.

#### Properties & Visual Configurations

- **`sortable`**: Opt a column into interactive sorting. Leave it unset for purely descriptive columns (label only); set it when the column's values should be reorderable by the user.
- **`direction`**: The reflected sort state of the column (`none`, `ascending`, `descending`). It is mutable and self-managed by the click cycle, but can be set externally — for example to clear a column to `none` — and drives both the rendered sort icon and the `aria-sort` value.
