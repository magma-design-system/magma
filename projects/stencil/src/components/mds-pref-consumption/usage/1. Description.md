The `<mds-pref-consumption>` web component is a preference control of the Magma Design System that lets the user pick an energy-consumption mode (`low`, `medium`, `high`); it is a compound child of [`<mds-pref>`](../../mds-pref) and renders internally as an `mds-tab` with one `mds-tab-item` per mode, so it has no native HTML primitive equivalent.

#### Semantic Behavior

- **Compound child constraint**: Must be placed as a direct slot child of `<mds-pref>`, alongside the other `mds-pref-*` controls; it is never used standalone or mixed with unrelated child types, because `<mds-pref>` discovers it by tag name to wire its events and propagate `size`.
- **Mode resolution on render**: On every `componentWillRender` the active mode is resolved in priority order — the `mode` prop, then the persisted `mdsPrefConsumption` value in `localStorage`, then the `high` default — so the control restores the last user choice across reloads.
- **Global side effects**: Selecting a mode persists it to `localStorage`, swaps the `pref-consumption-low|medium|high` class on the `<html>` element, and sets the `--magma-pref-consumption` custom property on the document root; this is how the preference actually takes effect page-wide.
- **Event bubbling to parent**: Each change emits `mdsPrefChange` with `{ preference: 'consumption' }`. The parent `<mds-pref>` listens (in controller mode) and, because `consumption` requires a reload to fully apply, surfaces its "reload required" notice.
- **Selection mirrors the active mode**: The three `mds-tab-item`s set `selected` from the current `mode`, so exactly one tab reflects the resolved preference; activation is handled via the tab items' click handlers.
- **Localization**: Exposes an `updateLang()` method and renders a localized caption label describing the current mode; the parent calls into language updates when locale changes.

#### Properties & Visual Configurations

- **`mode`**: The selected consumption preference (`low` / `medium` / `high`). Leave it unset to let the component restore from `localStorage` or fall back to `high`; set it explicitly only to force an initial mode. It is mutable and reflected, and a `@Watch` re-applies all global side effects whenever it changes.
- **`size`**: Sizes the nested tab items (`sm` / `md`). In normal use you do not set this directly — the parent `<mds-pref>` propagates its own `size` down to every `mds-pref-*` child, keeping the whole preference group visually consistent.
