The `<mds-input-upload>` web component is the Magma Design System file-upload control. It wraps a hidden native `<input type="file">` in a drag-and-drop drop zone with progress, per-file previews, validation and optional sorting, exposing the selected files to a surrounding `<form>` through form association.

#### Semantic Behavior

- **Form association**: `formAssociated: true`. The component mirrors the accepted files into its `ElementInternals` form value, so inside a `<form>` it submits like a native file input with no extra wiring.
- **Validity reporting**: Per-file errors are translated into native `ValidityState` flags — too many files sets `rangeOverflow`, oversize sets `tooLong`, wrong type sets `typeMismatch` — and the joined messages become the validation message via `setValidity`.
- **Form reset**: `formResetCallback` clears every selected file when the host form is reset.
- **Drag-and-drop**: The drop zone reacts to dragenter/dragover/drop; on dragenter the prompt text swaps to a drag hint and animates, restoring on dragleave.
- **Per-file validation**: Each added file is checked against `accept` (MIME, wildcard MIME, or extension), `maxFileSize`, and `maxFiles`; valid files render as success previews, invalid ones render as error previews carrying a localized message and are excluded from the form value.
- **Change event**: Emits `mdsInputUploadChange` with the current valid `FileList` (or `null`) on every add, cancel, or reset.
- **Imperative API**: Exposes `getFiles()`, `getFilesError()`, `reset()`, and `updateLang()` methods for host-driven control.
- **Localization**: All UI strings resolve through the shared `Locale` helper against the host language (el / en / es / it).
- **Composition**: Internally composes `mds-button`, `mds-text`, `mds-icon`, `mds-progress`, `mds-file-preview`, and (when sorting is active) `mds-tab` / `mds-tab-item`; it has no slots and is configured entirely through props.

#### Properties & Visual Configurations

- **`accept`** declares the allowed file types as a comma-separated list of MIME types, wildcard MIME types (e.g. `image/*`), or extensions; it both filters the native picker and drives the human-readable extension hint shown to the user.
- **`maxFileSize`** caps the size of any single file in MB (default `20`); files above it are rejected with a size error.
- **`maxFiles`** caps how many files may be uploaded (default `1`) and toggles the native input's `multiple` attribute when greater than one.
- **`initialValue`** seeds the control with files already present; it is watched, so reassigning it re-runs the add pipeline.
- **`sort`** controls the ordering widget: when omitted the user picks the sort and the choice persists in `localStorage`; when set to `'date'` or `'status'` it forces that order and hides the chooser. The sort tab only appears when `sort` is unset and more than one file is present.
