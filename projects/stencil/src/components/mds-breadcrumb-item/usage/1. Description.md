The `<mds-breadcrumb-item>` web component represents a single navigable depth in a breadcrumb trail. It is a compound child of [`<mds-breadcrumb>`](../../mds-breadcrumb) and renders an internal `<mds-button>` followed by a separator arrow icon, replacing the plain link an `<a>` would otherwise provide inside a navigation breadcrumb.

#### Semantic Behavior

- **Compound child only**: Must be placed as a direct default-slot child of `<mds-breadcrumb>`; it is never used standalone and the parent's slot should contain only `mds-breadcrumb-item` elements, not mixed child types.
- **Parent-assigned identity**: On load the parent assigns each item a positional `id` (`item-0`, `item-1`, …); the item itself does not own its index, so authored `id` values are overwritten.
- **Selection is parent-orchestrated**: Clicking the item toggles its own `selected` and emits `mdsBreadcrumbItemSelect` with `{ id, selected }`; the parent listens, then sets `selected` exclusively on the matching item and clears it on all others, so only one item is selected at a time.
- **Selected item is inert**: While `selected` is true the internal click/keyboard behavior is detached, so the current depth cannot be re-activated; non-selected items re-attach it.
- **Keyboard activation**: The internal text element is wired through the shared `KeyboardManager`, so the item responds to keyboard activation in addition to pointer clicks, with focus/blur outlines driven by the `--mds-breadcrumb-item-outline-*` custom properties.
- **Separator icon is decorative**: The trailing navigate-next arrow is rendered with `aria-hidden="true"` and carries no semantic meaning.

#### Properties & Visual Configurations

This child exposes essentially no visual configuration of its own — appearance (variant, tone, sizing of the underlying button) is inherited from `<mds-breadcrumb>` and the shared button system, and styling overrides are done through the `--mds-breadcrumb-item-*` CSS custom properties listed in the readme.

- **`selected`**: Marks this item as the current depth. It is reflected and mutable, but in normal use you set it on at most one item as the initial state; thereafter the parent manages it in response to navigation, so avoid setting it on multiple items at once. Use it only to seed which depth is active on first render.

The `label` slots the visible text into the internal button. Per the slot contract, the default slot should receive a plain text string only — do not place HTML elements or other components inside it, as that breaks the breadcrumb layout and compound communication.
