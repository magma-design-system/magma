The `<mds-filter-item>` web component is a single selectable option inside the [`<mds-filter>`](../../mds-filter) container of the Magma Design System. It renders as an interactive `menuitem` (icon, label, and an optional count badge) and acts as the toggleable unit that drives the parent's filter selection â€” it has no standalone HTML primitive equivalent.

#### Semantic Behavior

- **Compound child only**: `<mds-filter-item>` must be a direct slot child of `<mds-filter>`; it is never used standalone and should not be mixed with other child types, since the parent discovers items via `querySelectorAll('mds-filter-item')` and assigns each one a sequential `id` (`item-0`, `item-1`, â€¦) on load.
- **Accessibility role**: The host renders with `role="menuitem"` (the parent is the `menubar`) and an `aria-label` derived from `label`, falling back to `icon`; the nested icon/text/count are marked `aria-hidden` so the accessible name stays clean.
- **Self-toggling + event bubble**: Clicking (or activating via the keyboard) flips `selected` and emits `mdsFilterItemSelect` with `{ id, selected }`. The parent listens for this event to enforce single vs. multiple selection, scroll the item into view, mark adjacent `sibling` items, and re-emit `mdsFilterChange` with the aggregated value list.
- **State owned with the parent**: `selected` is `mutable` and `reflect`ed, so it doubles as a styling hook and as the flag the parent reads/writes â€” in single-select mode the parent clears the other items' `selected`, and on reset it sets `selected = false` across the group.
- **Keyboard activation**: A shared `KeyboardManager` attaches click behavior on load (and detaches on disconnect), so the item responds to keyboard activation like a pointer click.
- **Focus management**: The host is focusable (`tabindex="0"`) unless `disabled`, in which case it drops out of the tab order (`tabindex="-1"`).

#### Properties & Visual Configurations

- **`selected`**: The selection state of the item. Set it on initial markup to pre-select a filter; at runtime it is normally driven by the parent rather than set directly. Its reflected attribute also toggles the selected styling of the `count` badge.
- **`value`**: The form/data value contributed to the parent's aggregated `mdsFilterChange` payload when the item is selected. Provide a meaningful value distinct from the human-readable `label`.
- **`count`**: An optional badge showing how many results the filter would yield. Use it only when the count is meaningful to the user; omit it for purely categorical filters.
- **`icon`**: Optional leading icon (an icon name string). It also serves as the `aria-label` fallback, so an icon-only item without a `label` should use a recognizable icon.
- **`disabled`**: Renders the item non-interactive and removes it from the tab order; the parent uses this for the reset affordance when no filter is active.
