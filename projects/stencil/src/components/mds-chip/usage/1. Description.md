The `<mds-chip>` web component is the Magma Design System's compact element for representing a discrete piece of information, a filter, a selection, or a removable token. It renders a label with optional leading icon and an optional delete affordance, and can be made interactive (clickable or selectable) to act as a lightweight button-like control rather than a static tag.

#### Semantic Behavior

- **Static by default**: With no interaction props the chip is a passive, non-focusable label; it gains a `role="button"` on its label and a tabbable entry only when `clickable` is set.
- **Selectable implies clickable**: Setting `selectable` automatically turns on `clickable` via a watcher, so a selectable chip is always keyboard- and pointer-interactive.
- **Selection toggle**: When `selectable`, activating the label flips `selected`; a deselected chip normalizes `selected` back to `undefined` (never the literal `false`) so the reflected attribute is dropped rather than left as `selected="false"`.
- **Keyboard activation**: The interactive label is wired through the shared keyboard manager so Enter/Space activate it just like a pointer click, with focus handled via the label's `tabindex="0"`.
- **Delete affordance**: When `deletable`, a trailing `mds-button` is rendered with a localized, label-aware title and emits the delete event on activation; its title text is translated (el/en/es/it).
- **Disabled state**: Reflects `aria-disabled` on the host to convey the disabled status to assistive technology.
- **Emitted events**: `mdsChipClickLabel` fires on label activation for non-selectable chips, `mdsChipSelect` fires (carrying the new `selected` value) for selectable chips, and `mdsChipDelete` fires from the delete button — all `CustomEvent<MdsChipEvent>` carrying the originating event and host element.
- **Label is text, truncated**: The label renders inside `mds-text` with `truncate="word"`, so long content is clipped rather than wrapping.

#### Properties & Visual Configurations

The shared `variant` / `tone` ladders are defined in [`projects/stencil/SPEC.md`](../../../../SPEC.md#tone-and-variant-system); the chip defaults to `variant="primary"` and `tone="strong"`, and accepts only the minimal `'strong'` / `'weak'` tone set.

#### Other behavioral props

- **`clickable`** opts the chip into interactivity (focus, `role="button"`, keyboard and click handling) without changing its state on activation — use it for chips that act as actions rather than toggles.
- **`selectable`** makes the chip a toggle whose `selected` state flips on each activation; pick it for filter/choice chips where the visual selected state matters.
- **`selected`** drives the selected visual treatment and can be set externally to pre-select a chip; it is normalized so an unselected chip carries no attribute.
- **`deletable`** adds the trailing cancel button for removable tokens; reserve it for chips the user is meant to dismiss.
- **`icon`** is an SVG filename slug from the Magma icon library, shown (decoratively, `aria-hidden`) to the left of the label.
