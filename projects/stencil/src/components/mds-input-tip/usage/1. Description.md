The `<mds-input-tip>` web component is the floating helper container of the Magma Design System input family: it overlays an input field and reveals supplementary content (slotted `mds-input-tip-item` chips, hints, or actions) that expands when the field becomes active. It is a presentational compound child positioned absolutely against its parent input rather than a standalone control.

#### Semantic Behavior

- **Absolutely positioned overlay**: The host is `position: absolute` with `pointer-events: none`, so it floats over the parent input's edge without intercepting interaction; slotted children opt back into pointer events as needed.
- **Active expansion**: Setting `active` reflects to the host and applies a transform that slides the tip into its expanded position, animated via the system motion tokens; clearing it collapses the tip back flush with the input.
- **Empty collapse**: When the host has no slotted content (`:host(:empty)`) it is hidden entirely, so an unused tip occupies no space.
- **Border seam with items**: While active, slotted `mds-input-tip-item` elements have their radius flattened on the edge adjoining the input (top corners for `position="top"`, bottom corners for `position="bottom"`) so the tip reads as a continuous extension of the field.
- **Default slot is content**: The single default slot carries the tip's arbitrary children; there are no named slots, events, form association, or focus management of its own.

#### Properties & Visual Configurations

- **`active`** drives the expanded vs. collapsed state and is normally toggled by the parent input in response to focus, not set manually.
- **`position`** selects which edge of the input the tip anchors to and therefore the direction it expands: `'top'` (default) sits above the field, `'bottom'` sits below it. The choice also determines which item corners are flattened during expansion.
