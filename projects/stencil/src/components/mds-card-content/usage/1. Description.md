The `<mds-card-content>` web component is the body region of a [`<mds-card>`](../../mds-card), grouping the primary textual or component content of a card. It is a thin layout wrapper with no HTML primitive equivalent: it exposes a single default slot and assigns itself to the parent card's `content` slot automatically.

#### Semantic Behavior

- **Compound child only**: `<mds-card-content>` must be placed as a direct child of `<mds-card>`; it is never used standalone, and a card's content region is expected to be a dedicated `mds-card-*` child rather than mixed loose markup.
- **Automatic slot targeting**: The host renders with `slot="content"`, so it lands in the parent's named `content` slot without the developer having to set the `slot` attribute manually.
- **Participates in the parent layout signature**: Because its tag is prefixed `MDS-CARD-`, the parent's `componentWillLoad` detects it and folds a `content` token into the card's computed `layout--*` grid class, which is how the card adapts its responsive container-query layout to the regions actually present.
- **Internal grid container**: The host is itself a CSS grid with auto-fit columns, spacing, and padding, so multiple slotted children stack with consistent gaps and the region never overflows its track (`min-width: 0`).
- **No state, events, or ARIA**: The component declares no props, state, watchers, or emitted events and applies no role or ARIA attributes; it is purely structural and inherits semantics from whatever is slotted into it.

#### Slot semantics and layout role

This component is intentionally prop-free. Its only API is the **default slot**, which accepts text strings, HTML elements, or other components that make up the card's main content. All visual configuration (responsive behavior, gap, padding) is governed by the parent `<mds-card>` through its `autoGrid` prop and the `--mds-card-gap` / `--mds-card-padding` custom properties; see the compound-component and slot rules in [`projects/stencil/SPEC.md`](../../../../SPEC.md).
