The `<mds-input-date>` web component is the Magma Design System control for capturing a single calendar date. It wraps a native `<input type="date">`, adding form association, ISO value validation against an optional `min`/`max` range, contextual tips, and a button-triggered calendar dropdown for visual date picking.

#### Semantic Behavior

- **Form association**: `formAssociated: true`. The host participates natively in form submission and exposes its `value` under `name`; on form reset it clears the submitted value.
- **ISO value contract**: `value`, `min`, and `max` are all ISO date strings (`YYYY-MM-DD`). The component parses them with Luxon and never accepts other formats.
- **Validation on change**: every value change runs validation and emits `mdsInputValidation` with a boolean. When the date is invalid (and `required`) or falls outside the `min`/`max` range, the component forces `variant` to `'error'` and submits `null` to the form; otherwise it restores `'primary'` and submits the value.
- **Range self-correction**: if `max` is earlier than `min` at load, `max` is snapped to equal `min`.
- **Selection event**: emits `mdsInputDateSelect` (bubbling, composed) with the new string value whenever `value` changes, whether typed or picked from the calendar.
- **Calendar dropdown**: the trailing calendar button opens an `<mds-dropdown>` hosting a single-date `<mds-calendar>`; picking a day writes back the value and, after `delay`, auto-closes.
- **Slotted mode**: when the host carries a `slot` attribute it is treated as embedded — the calendar button, dropdown, and calendar are not rendered, leaving only the bare input for composition inside a larger field.
- **Contextual tips**: an `<mds-input-tip>` surfaces `disabled`, `readonly`, and `required` states; the required tip expands on focus and reflects success once the value is valid.
- **Focus & read-only**: focusing tracks `hasFocus` for the tip; a read-only field auto-selects its text on focus instead of allowing edits.

#### Properties & Visual Configurations

The shared `variant` ladder is defined in [`projects/stencil/SPEC.md`](../../../../SPEC.md#tone-and-variant-system). Note that `variant` is mutable here: the component overrides it to `'error'` / `'primary'` as validation dictates, so an externally set variant is not authoritative once the user interacts.

#### Other behavioral props

- **`min`** / **`max`** bound the selectable range; dates outside it invalidate the field rather than being silently clamped.
- **`delay`** is the milliseconds to wait before auto-closing the calendar dropdown after a pick; setting it to `0` keeps the dropdown open.
- **`required`** makes an empty or invalid value fail validation and drives the required/required-success tip.
